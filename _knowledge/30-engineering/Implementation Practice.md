# Implementation Practice

Implement only approved acceptance criteria. Preserve repository conventions and pre-existing work. Prefer small, reviewable changes. Validate inputs, authorization and failure handling at trust boundaries.

Every change hands off revision, files, checks, limitations and residual risks to [[Testing Strategy]]. Security-sensitive changes also route to [[Security Map]].
