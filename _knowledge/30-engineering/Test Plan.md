# Test Plan

A plan maps profile checks to configured scripts, expected scope, required state and command preview. It does not execute checks.

Mandatory unavailable checks block readiness. Optional unavailable checks remain visible as `not_configured`.

Tool: `testing.plan` in [[Testing MCP]]. Related: [[Testing Strategy]], [[Evidence Model]].
