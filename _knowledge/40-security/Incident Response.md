# Incident Response

1. Validate the signal and establish incident owner.
2. Preserve timestamps, revision, logs and affected resources without spreading sensitive data.
3. Contain within approved authority.
4. Assess identity, secrets, data and availability impact.
5. Eradicate root cause and rotate affected credentials through approved systems.
6. Recover with health and security verification.
7. Document timeline, decisions, notifications and follow-up actions.
