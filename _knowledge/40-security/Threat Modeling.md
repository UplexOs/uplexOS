# Threat Modeling

Document assets, actors, privileges, entry points, trust boundaries, data flows, external processors, abuse cases, controls and residual risk. Choose STRIDE, misuse cases or attack trees according to scope.

Threat models should produce testable security requirements and link to [[Testing Strategy]], [[Risk Management]] and [[Security Gate]].
