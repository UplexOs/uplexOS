# Language policy

- Follow operational instructions in English.
- Respond to the user in Brazilian Portuguese by default.
- Preserve commands, paths, identifiers, diagnostics, package names, advisories and source code in their original language.
- Produce customer-facing artifacts in the active project's configured locale.
- Do not translate tool output in a way that changes technical meaning; summarize it separately in the output locale.
- If the requested locale is ambiguous, use `pt-BR` and state the assumption only when it materially affects the deliverable.
